Member Firm Website Program

M Financial has over 100 financial advisor Member Firms, and nearly all of them had very outdated websites in need of modernization. The trick was to find some way to create fully branded and unique websites for each firm at a fraction of the cost they would pay on the open market.

My solution was to create a multi-concept Wordpress theme that provides option controls for nearly every aspect of the design. This enables us to quickly and easily adapt each website to the firm’s unique branding and vision at very low cost and quick turnaround. We manage all the websites in a Wordpress Multisite instance which allows us to keep all of them up-to-date and secure. Custom plugins I have developed allow the firms to leverage a compliance-approved content library, and streamline their requests for support and compliance reviews.

Finally, all of this is cloud-hosted on DigitalOcean, using a network of VPS servers that I stood up and configured, highly available with 99% uptime.

Parkzilla

Founder Tim Ashman came to Harlo with unique idea to create an app that would allow users to offer their driveway or parking space for rent. The app would have iOS and Android versions, as well as a web-based profile management tool.

In order to accommodate all three platforms, I decided to create a cloud-based API using Parse.com’s SaaS. I used Parse's javascript platform (based on Backbone.js) to build a booking engine and Stripe-integrated marketplace, with API endpoints and corresponding documentation. I worked closely with the mobile app developers to tailor the API to their needs.

I then built out the web application using the Laravel PHP framework that allows users to register, connect with Stripe, and set up their parking spaces for rent with custom pricing schedules.

Elemental CMS

I built this platform in-house for Harlo. We needed a streamlined way to create custom CMS solutions for clients while also having a ready-made solution to shorten development timelines. Elemental is the best of both worlds.

Elemental leverages many of Laravel 5's modern PHP features, including namespaces, interfaces, service provider and facade patterns, and the Eloquent ORM. I also developed a command line interface using Laravel's Artisan Console tool. I designed this architecture to make it easy for my development team to install Elemental as a Composer package and get up and running. All of the heavy lifting is abstracted away and developers have only to create some basic array structures to define their CMS components. They can then generate the CMS user interface with a few simple commands in the CLI.

Elemental also exposes a REST API to interact with the Angular front-end services, as well as any other third-party integrations as needed. Elemental's front-end UI wraps the Semantic UI library in custom Angular directives.

Finally, I wrote clear and concise documentation to help onboard new team members quickly.

Zendesk for M Financial

M Financial approached Vogey to help them select and implement an enterprise ticketing system that would provide the flexibility to meet the complex business needs of several different internal departments.

After an in-depth discovery phase and tools assessment, Zendesk was selected and we began implementation. M now utilizes Zendesk's flexibility and features to the fullest extent, providing tech support to it's Member Firms, internal service desk ticketing, automated communication and marketing compliance workflows, and as an advisor reporting platform that meets FINRA books and records requirements.

My role was to help discover each department's unique business needs and then implement best practice workflows using Zendesk's triggers, automations, views and ticket forms. I developed custom Zendesk apps that help the compliance team approve and capture marketing material updates and streamline advisor reporting. Using Zendesk's API, I created a risk-based algorithm to sift through large volumes of financial reporting tickets and flag a percentage for review. I partnered closedly with M's IT team to implement single sign-on, sync user and organization data, and bulk export Zendesk ticket data into third-party tools.

Finally, I helped deliver value on the platform to M's upper management by building custom GoodData reports around their specific KPIs and metrics.

Gesundheit Foods ERP

Gesundheit Foods approached Vogey to help them select and implement Enterprise Resource Planning software that would help them create efficiencies, reduce costs, and scale quickly.

The largest cannabis-infused food and beverage co-packer on the West Coast, Gesundheit Foods is growing quickly from their Portland headquarters and building out additional facilities in Oakland, LA and Vancouver, B.C. Their rapid growth had left them without a good integrated solution for the administration of their inventory, shipping, R&D and supply chain processes.

After an intensive discovery phase to learn about their business workflows and pain points, I researched dozens of ERP platforms, from open source to mid-market and enterprise solutions like SAP and NetSuite.

Ultimately I settled on a Sage X3 implementation that includes various canna-business modules developed by NexTec Group. I worked closely with the NexTec team to put together a proposal that would meet Gesundheit Food's highly specific needs.

On the technical side, I helped put together a hosting solution on AWS for the Sage X3 cloud-based platform. I am currently working to integrate Sage X3's API with the state-required cannabis regulation platform Metrc, which will eliminate hours of manual data entry.
